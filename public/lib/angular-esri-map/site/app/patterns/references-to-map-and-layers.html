Getting a Reference to the Map and Layers

We've provided a couple of ways for you to get direct references to the map and layer objects from a parent controller. This will allow you to manipulate those objects in ways beyond what is allowed by the directive API.

Load Events

The map and layer directives each include a load event that passes a reference to the underlying map or layer object to a callback function. See these examples:

  • Map Events
  • Layer Events

Registry

We've also included a registry service that you can use to get a direct reference to a map object by setting the register-as attribute on the map directive. See the Registry Pattern example.
